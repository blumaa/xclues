# SEO Archive-Hub Plan (for tomorrow)

Goal: turn the ~690 already-generated archive pages from **sitemap-only orphans** into an
internally-linked content graph Google can crawl, rank, and surface. This is the single
highest-impact organic-traffic lever right now (the SEO foundation — sitemap, robots,
metadata, OG images, SSR content — is already solid).

> Owner has more ideas to add before we start — leave room. This is a starting skeleton.

## The problem (evidence)
- `app/sitemap.ts` lists every published archive page (`/[genre]/archive/[date]`), ~690 URLs,
  auto-growing ~3/day.
- BUT there is **no `/[genre]/archive` index page** and **no prev/next links** between daily
  puzzles. Grep found zero `archive/` links in any component — they exist only in the sitemap.
- Orphan pages (crawlable but not internally linked) rank far weaker. We're leaving the
  archive's ranking potential on the table.

## Proposed work (TDD — write the failing tests first)

### 1. Archive index hub — `/[genre]/archive`
- New route listing all published dates for that genre, newest first, **paginated** (e.g. 30/page
  with `/[genre]/archive?page=2` or `/[genre]/archive/page/[n]` — decide which is cleaner for SEO).
- Server-rendered, each row links to `/[genre]/archive/[date]` with descriptive anchor text
  (date + puzzle theme/headline if available, not just a bare date — better for relevance).
- `generateMetadata`: title/description/canonical per genre (mirror existing archive-date page).
- Reuse `fetchAllPublishedDates()` (already used by the sitemap — SSOT).

### 2. Internal links INTO the hub
- Footer (and/or header) link to "Archive" on every page → gives the hub link equity and a crawl path.
- Link from each `/[genre]/archive/[date]` page back up to the archive index (breadcrumb).

### 3. Prev/next day links on each archive-date page
- On `/[genre]/archive/[date]`, add "← previous puzzle / next puzzle →" linking adjacent published
  dates. Creates a complete crawl chain through all ~690 pages + helps users browse.

### 4. BreadcrumbList structured data
- Add `BreadcrumbList` JSON-LD to archive pages (Home › Archive › [date]). Pairs naturally with the
  breadcrumb UI from #2 and can surface breadcrumbs in SERPs. (Currently archive has `WebPage`
  schema only.)

## Also worth folding in (cheap, related — from the audit)
- **Search Console:** confirm all 3 domains added + each sitemap submitted (DNS verify is fine).
  Without it we're blind on indexing coverage. Free, do first.
- **Delete stale static files:** `public/sitemap.xml` (only 3 URLs) and `public/robots.txt`
  (generated by `scripts/generate-seo.ts`, only for the CAPACITOR prebuild). Live serves the
  dynamic `app/` versions, so they're harmless today — but a static `public/` file can shadow an
  `app/` metadata route on a future deploy. Remove the footgun.
- **Per-domain sitemap scoping (minor):** each domain's sitemap currently lists cross-genre URLs
  (e.g. filmclues.space sitemap includes `/music`). Canonical handles it, but trimming to own-genre
  only reduces crawl noise. Low priority.
